# Regra: Não Criar Documentos de Análise

## Comportamento Obrigatório

Quando o usuário pedir análises, sugestões, melhorias ou revisões:

- **NUNCA criar arquivos .md de documentação**
- **SEMPRE explicar diretamente no chat**
- Ser conciso e direto
- Usar bullet points quando necessário
- Focar no que importa

## Exceções

Criar documentos APENAS quando:
- O usuário explicitamente pedir "crie um documento"
- For parte de uma spec formal
- For documentação técnica de código (README, API docs, etc)

## Exemplos

❌ **ERRADO:**
- Criar `DASHBOARD_ANALYSIS.md`
- Criar `IMPROVEMENTS_SUGGESTIONS.md`
- Criar `REVIEW_REPORT.md`

✅ **CERTO:**
- Explicar tudo no chat
- Mostrar código inline
- Listar melhorias diretamente
